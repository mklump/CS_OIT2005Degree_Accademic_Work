Team Deliverable for Alpha Release

Due Date : Tuesday April 27, 2004

Objective:   Complete first stage of project implementation

What to do:  

  • Project Schedule completed (Larry)
  • List of Requirements implemented (Mike) 
  • List of Use Cases implemented (Gary)
  • Test Plan completed (Everyone)
  • A Test Case Package (Everyone)
    • Define necessary test environment.
    • For Each Use Case have at least one test case.
  • Make your program and test cases accessible by the team that will be testing it.

How to do it:  

  • Make sure your web pages are up to date and it is easy to locate the necessary deliverables (Larry)

  • You should have a "project" schedule completed.  This schedule should detail the work (tasks) to be completed, the order that they should be completed, who will do the work, and how long it should take to complete it.  I'd suggest that the individual responsible for the task also write the test case for the unit of work, if appropriate.

  • You should have a Test Plan from the last term.  Review it for completeness and update as necessary.  The test plan is just that - a plan on what you're going to test.  It should define how you are going to test your project, a list of the test cases, and in what order the testing should be done.

  • You should complete a test case for each use case or feature that you have implemented up to this point.  Ideally, you would have most of the core components implemented and unit tested.

  • Remember that this is Alpha quality code.  Do not expect bug-free applications, but do detail how to achieve that in your schedule (i.e., detailed work load breakdown and adequate testing time). 

  • If reasonable, make your program and test cases available for another team to test.
