Blues Home Page
  

Student login page:

 

No active code. Just HTML containing a field for the key to be entered in and a submit button that sends the input text to the next page (in addition to some graphics and junk)

 

Course evaluation page:

 

Receive key

Access DB to see if key exists

If key does not exist:

  display error and exit.

 

If key does exist

  Mark key as in use for next 30 minutes

  access DB an get all evaluation questions associated with the key

  Display each evaluation question in HTML with appropriate answer fields

  Present submit button to page responsible for storing the evaluation

 

Evaluation storage page:

 

Check all questions on the passed evaluation for violations of preset criteria (question not answered, valid key, etc…)

If evaluation passes check:

  Store responses in DB for retrieval later

  Destroy key in DB used to access evaluation to prevent duplicate access

  Present the student with success message. Exit system.

 

If evaluation does not pass check:

  Present student with reason why and optional solutions (depending on what went wrong).
