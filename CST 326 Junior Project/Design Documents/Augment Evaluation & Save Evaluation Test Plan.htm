Test Plan for the Augment Evaluation Web Page Module

By Matthew Klump

March 13, 2004

  1. Introduction

The Augment Evaluation and Save Evaluation web pages are meant to provide the structure for which will be contained the feature of OwlEval that will allow Administrators and Faculty the ability to add additional questions to the standard set of questions that the students will answer for each course evaluation.

To our great pleasure, this feature was requested specifically by the OIT faculty and administration. It is an honor to include it for this version of the online course evaluation system that will replace the currently over burdensome paper based system.

  1. Relationship to other documents

We have broken the OwlEval system into five components:

 

1.      Student evaluation completion

2.      Faculty login and reports viewing

3.      Evaluation augmentation

4.      Evaluation creation and key generation

5.      Database access

This is the third module of five in all that will fully implement the OwlEval online course evaluation.

 

  1. System overview

In the grand scheme of the OwlEval web design, Augment Evaluation is the second webpage that can be navigated to from the Administrator/Faculty menu page. The complete Augment Evaluation Module comprises of the webpage to enter and to review the additional questions to add with the standard questions for saving to the database (Save Augmentation).

  1. Approach

(Comment credit to Glenn Diviney)

To test these features, the basic page framework will be constructed with external variables assumed and proper. Each feature will be constructed and verified with all feasible inputs in an attempt to crash the system.


  1. Suspension and resumption

Since each page must complete processing before it is displayed to the user (because it will be written in PHP), any error condition must be noted and displayed as part of the normal page processing routine. Suspension and resumption will generally be handled in this way.

  1. Testing materials (hardware/software requirements)

One PC with PHP and MySQL installed locally.
