Individual Review and Testing of Alpha release

Due Date:  Tuesday May 4, 2004

Review another Project team's Alpha Release
Review the project schedule and write up a brief review.  

  • does the schedule look "doable" in the time allotted
  • are all the feature/functions included
  • are the resources identified for each task
  • does the schedule include complete test coverage tasks
  • etc.

Review the project test plan and write up a brief review.  

  • is there reasonable time allocated for testing
  • does the test plan cover each use case and requirement
  • are there test cases for unit testing, blackbox testing, whitebox testing, integration testing, etc.
  • etc.

For each test case defined by the team's project you are testing

  • Submit a completed test report
  • Submit a test case critique:
  •  
    • How useful were the test cases?
    • Do you think they performed the intended purpose?
    • Rate the Test Cases defined on a scale 1-10, 10 being highest.

What to hand in for Individual Alpha Review
 Be certain to include at the top of the page your name, the project name and project deliverable you are reviewing.
Make this a constructive review document that will aid the project team to be more successful and avoid pitfalls and oversights.

  1. Write up 1-2 paragraph assessment of the reviewed project's schedule.  
  2. Complete a test report that includes a review of the project test plan and each test case that was completed.  Can you easily trace the test case to the use case and the defined project requirement?  Is the test case complete, i.e., does it really test edge cases and error paths?
  3. If possible, execute each test as defined in the test case and record the results.  You can use the form provided under "Forms" menu.
