Team Deliverable for Beta Release

Due Date : Tuesday May 11, 2004

Objective:   Complete second stage (Beta) of project implementation

By now, you should be at "functionality complete" or close to it, and entering the integration and installation stage.  Ideally, all of the defined features/functions have been implemented and tested.  You should be completing the final round of testing and, as appropriate, completing the installation and documentation work.

What to do:  

  • Complete the mapping of Requirement to Use Case to Test Case  ( see sample below )
  • Complete the Test Case Package
    • Define necessary test environment.
    • For each remaining Use Case have at least one test case.
  • Make sure your web page is up-to-date for an audit.
  • Make your program and test cases accessible to the team that will be testing it.

How to do it:  

  • Make sure your web pages are up to date and it is easy to locate the necessary deliverables

  • You should have an updated "project" schedule with a jogging date line.  This schedule should detail the work (tasks) already completed, and the work still remaining to be completed.

  • Update the test plan as necessary.  

  • You should complete a test case for each use case or feature that you have implemented up to this point.  Ideally, you would have all of the tasks implemented and unit tested.
