Senior Project FAQ

Project Objectives

 

Ask yourself the following questions about your project objective:

 

  1. How is this different from existing products?  For example if my project objective statement was:

Create an online mortgage and loan calculator.

 

I would ask you:  How is this different from http://www.mortgagestart.com/calculator.html or any of the many other loan calculators that are currently available.
  1. If you are doing a project that maintains and administers data ask yourself:  What is special about this project that cant be done in Excel or Microsoft access?

 

  1. Can this project be used for 412, 422 and 432?  For example, a porting project would probably not be appropriate for a senior project because there isn’t a lot of design work required.  (ie work from 412)  .  (The design for a porting project is frequently just “keep it the way it is” or “don’t break it.”.)

 

Questions and Answers:

Question:    Can I upgrade an existing system for my senior project?

Answer:      Probably not.  If you are rewriting an existing system in a new language / environment, it may be ok if you are willing to start from scratch.  (See Item 4 above)

 

Question:    Can I use programming language XYZ or operating system ABC?

Answer:      Refer to the CST334 Course of Study for department requirements.  Other languages will be evaluated on a case by case basis.  Keep in mind that you will be required to demo your project at various times during the year, and I will need to be able to evaluate / test the end result.  If you want to use systems other than those available in the OIT you may need to loan a system to me for an extended period of time.

 

Question:    Can I do a web site.

Answer:      No.  You can however, do an application that is used on a web site.

 

Question:    Can my project be an add-on to an existing application?

Answer:      No.  You should create a stand-alone application.  You can potentially have output to other applications.

 

Notes:

  • Is-Is Not statements are not usually technical specifications.  They give the user an idea what they are buying.  An end user doesn’t usually care if you programmed it in Java or zobalo.
  • If you are not a domain expert in your project area, you need to find a sponsor or domain expert that you can interview and bounce ideas off of.
