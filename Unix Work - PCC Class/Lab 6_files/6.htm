Chapter 12, Redirection and Piping

  1. Problems (p. 314), Question 12. What is the UNIX pipe? How is pipe different from output redirection? Give an example to illustrate your answer.

 

(a)   The UNIX pipe connects the standard out of a command to the standard in of another command.

(b)  The pipe plugs together two commands through stdin and stdout, while output redirection simply takes the command output from stdout, and redirects it to a file.

(c)   In the command ls –al | less 1> list.fullview , the pipe between ls and less connects the stdout of ls –al to the stdin of the less command, and the 1> between the less command and the file list.fullview redirects the stdout of less to the file list.fullview.

Chapter 13, Processes

  1. Problems (p. 346), Question 4. What are the main states that a process can be in? What does each state indicate about the status of the process?

 

(a)   The main states a UNIX process can be in are Ready, Running, Waiting, Swapped, and Zombie.

(b)  Ready indicates the process is ready o run, but does not have the cpu, Running indicates the process is actually running, Waiting indicates the  process is waiting for an event, Swapped indicates the process is ready to run, but it has been temporarily put on the disk, Zombie indicates a dying process that is not yet completely stopped.

  1. Foreground and Background Processes
    1. Look at the man pages for the sleep command. What does the command sleep 900 do?

 

                        The command sleep 900 will cause the terminal to pause for 900 seconds.

 

    1. How do you run a command as a foreground process?

 

                        Commands are executed in the foreground by default.

    1. How do you run a command as a background process?

 

                        You run a command as a background process by adding the ampersand ‘&’ as the last argument for the command you are using.

 

    1. How do you list all background processes?

 

                        You list all the background processes by using the à jobs [jobid]... command.

 

    1. How do you start a particular background process running (keeping it in the background)?

 

                        You start a particular background process running again while still in the background by using à bg [jobid]... command.

 

    1. How do you bring a particular background process into the foreground?

 

                        You bring a particular process to the foreground by using the à fg [jobid] command.

 

    1. How do you suspend the foreground process?

 

                        You suspend the foreground process by using the ^Z command.

 

    1. How do you suspend a particular background process?

 

                        You suspend a particular background process by moving it to the foreground à fg [jobid], then suspend it in the foreground with ^Z.

 

  1. Problems (p. 347), Question 9. In UNIX jargon, what is a daemon? Give examples of five daemons.

 

                  A daemon is a system process running in the background. Five examples of daemons are: syslogd, update, klogd, inetd, and rpc.mountd.
