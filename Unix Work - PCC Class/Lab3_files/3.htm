Chapter 9, Basic File Processing:

  1. Review the Unix commands covered in chapters 3, 4.3, 7, and 9 (not editors or email). On cs00, use your favorite text editor to create a reference guide using the format of the example given below.
    • You should have at least one entry for each command mentioned in these chapters.
    • Add entries for any other commands/options you feel are relevant.
    • Include separate entries for specific options that you think might be useful to you. Do not add an entry for each option presented in the book (e.g. you might use ls -l, rm -i, man -k regularly, but you will hardly ever use ls -i).
    • Use [ ] to indicate optional options/arguments.
    • Use ... to indicate that more than one of a particular argument may be supplied.
    • Make sure your reference guide is in alphabetical order by description (not command).
    • Be as specific as possible in your description, so that alphabetical order makes sense (e.g. use "remove a file" instead of "get rid of a file").

    Example: 

    UNIX COMMAND REFERENCE GUIDE

Description                               Command
--------------------------------------    ----------------------
Copy a file to a different name           cp oldfile newfile
Copy file(s) to a different directory     cp file... directory
List files in current directory           ls [file...]
List files in long format                 ls -l [file...]
Move file(s)                              mv file... directory
Remove file(s)                            rm [file...]
Remove an empty directory                 rmdir directory
Remove a directory (dangerous)            rm -r directory
Rename file                               mv oldname newname
