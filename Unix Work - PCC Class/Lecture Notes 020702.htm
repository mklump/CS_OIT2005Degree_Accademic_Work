Unix Shells (continued)

  • Shell Features
    • Set of available features and their richness varies from one shell to another
    • Command Line Editing: Editing of commands as they are typed
      • Can control the editor style:
        • set -o {emacs|vi} controls style in Korn, Bash
        • bindkey {-e|-v} in Enhanced C shell (tcsh)
    • Command History: Repeat/edit/list previously typed commands
    • Job Control: Manipulate child processes of current shell (chapter 13)
    • Filename Completion: Search for files in current directory matching what is typed
      • <Tab> in Bash and Enhanced C Shell
      • <Esc><Esc> in Korn
    • Aliases: Shortcuts, stored in memory, executed at will
    • Functions: Whole pieces of code, stored in memory, executed at will
    • Expansion/Substitution
      • What the shell does before a command is executed
      • All shells will perform certain substitutions to remain backward compatible with Bourne shell
      • Some of those substitutions (in the order they will be performed):
        • Tilde: E.g.: cd ~/bin or cd ~charliet
        • Parameter and Variable: E.g.: echo $HOME
        • Command: E.g.: ls -l `which bash`
        • Arithmetic: E.g.: echo $[ 2 + 3 ]
        • Pathname: E.g.: ls chapter*.txt
      • Some shells support additional substitutions which typically are performed first
      • For example:
        • Alias
        • Function
        • Brace: E.g.: ls chapter1.{eps,prn}
        • History: E.g.: cd; !397 -l
  • Command History
    • C Shell and Enhanced C Shell:
      • Event Designators
        • !N = Command number N
        • !-N = The Nth most recent command
        • !! = The most recent command (synonym for !-1)
        • !string = The most recent command starting with string
        • !?string? = The most recent command containing string
        • ^oldstring^newstring^ = Quick substitution on most recent command
      • History substitution:
        • Use an event designator anywhere in command line
        • The shell will replace it with the command from history
        • E.g.: If command number 5 was simply ls then, typing !5 -l will
          execute           ls -l
        • Similarly, typing echo !5 will execute echo ls
    • Korn Shell: Fix Command: fc [-l] [N [M]]
      • With no arguments starts editor with most recent command
      • Uses FCEDIT environment variable (if not defined, uses EDITOR)
      • Command is executed as soon as you exit the editor.
      • N argument specifies a specific command:
        • If N is a number, it means command number N in the history
        • If N is a negative number, it means the Nth most recent command
        • If N is a string, it means the most recent command starting with that string
      • M argument can be used to specify a range of commands, N through M
      • The same rules apply to M as those for N
      • -l = Just display the command, do not execute
        • If used with no argument, only the 16 most recent commands are displayed
      • history = Display entire contents of history (synonym for fc -l)
      • r N = Reexecute command N (synonym for C Shell's !N)
  • Shell Built-In Commands
    • Most commands are executable programs
    • Some commands are built-in to the shell, meaning they are part of the same program
    • Different shells have a different built-in commands
    • Commands may be built-in because they are:
      • used frequently (e.g. cd)
      • shell features (e.g. alias, history)
      • implemented differently by the current shell (rare)
        (e.g.         /bin/echo supports the --help and --version options, but Bash's built-in echo does not).
    • In some shells, you can use type command (or whence command) to determine whether command is built-in
    • To find out more about what commands are built-in to your shell, see man pages for your shell

Chapter 10. Advanced File Processing (picking up a few things we left behind)

  • 10.3 Compressing Files
    • compress -v file...
      • Replaces original file with compressed version
      • Appends .Z to filename
    • uncompress -v file...
      • Replaces compressed file with uncompressed version
  • Better Compression
    • gzip -9 file...
      • -9 means maximum compression
      • Appends .gz to filename
    • gunzip file...

 

  • 10.8 Encoding and Decoding
    • Converts binary files (containing non-printing characters) to ASCII encoded form (not human readable)
    • Useful for sending/receiving binary files (eg. programs, graphics) via email
    • Not used much anymore because MIME support built into most email readers
    • uuencode file label > encfile
      • file = file to be encoded
      • label = filename to use when file is decoded
      • encfile = file in which to store encoded data
    • uudecode encfile
      • encfile = file in which to encoded data is stored
      • Automatically creates file named label (specified during encoding)

 

  • 10.9 Encryption and Decryption
    • Useful for securing files so others (including root) cannot read them
    • Security warning: Files are vulnerable to attack
    • Encrypted files still contain "binary" (non-printing) characters
    • Encryption uses a "key", which is like a password
    • key is used for both encryption and decryption
    • crypt key < file > cryptfile
      • Encrypt file as cryptfile with key
    • crypt key < cryptfile > file
      • Decrypt cryptfile with key, and store output in file

Chapter 19. Backup Procedures

  • Archiving and Restoring Files Via tar
    • Archiving Files
      • tar -cvf archfile file...
        • -c = create new archive
        • -v = verbose (display information during archiving)
        • -f = store in archfile
      • Stores files and directories (recursive)
      • Does not compress contents
      • archfile typically has a .tar extension
    • Restoring Archived Files
      • tar -xvf archfile [file...]
        • -x = extract archive
      • Extracts files from archive
      • If file... is specified, only those files/directories will be extracted
    • Copying Directory Hierarchies
      • Ignore the book--use cp -rp if you want to copy a directory structure
      • Book says permissions and file mod times won't be preserved--that is what the -p is for.
      • Some older systems may not have -r. That means it is time for an upgrade.
    • Software Distributions in the tar Format
      • Files ending in .tar
      • Files ending in .tar.Z or .tZ
      • Files ending in .tar.gz or .tgz
